Sentiment Analysis with NLTK & HuggingFace Transformers

I used NLP to classify sentiment across 500,000+ Amazon product reviews, comparing two different approaches — NLTK's VADER model and a HuggingFace RoBERTa transformer. The goal was to see how well each one could pick up on whether a review was positive, neutral, or negative, and then check how that lined up with the star ratings people actually gave.

Notebook
Results

Across 500 Amazon reviews, both models performed well but told different stories. VADER's compound scores trended in the expected direction — lower for 1-star reviews and higher for 5-star reviews — but showed notable overlap between mid-range ratings (2–4 stars), where sarcasm and nuanced language challenged its rule-based approach. The RoBERTa transformer consistently produced sharper, more confident sentiment separations, particularly on reviews where the tone contradicted the literal word choice — something VADER's bag-of-words method struggled to detect.

Fake News Detection System

I built a machine learning pipeline to classify news articles as Fake or Real using their text content. I merged labeled datasets, cleaned and preprocessed the article text, vectorized it with TF-IDF, and then trained and compared four different classification models to see which ones held up best.

Notebook
Results

All four models performed exceptionally well on the held-out test set of over 11,000 articles. Logistic Regression and Random Forest both achieved roughly 98.8% accuracy, while the Decision Tree and Gradient Boosting classifiers edged slightly higher at around 99.5%, with near-perfect precision, recall, and F1-scores across both classes. On manual testing with unseen news articles, all four models agreed consistently — correctly flagging fabricated stories as Fake News and legitimate reporting as Real News.

04
Year-over-year revenue decline

Two CTEs total 2022 and 2023 revenue per branch, then join them to compute a revenue decrease ratio and surface the 5 branches that dropped the most.

Multi-CTE
WITH revenue_2022 AS (
    SELECT branch, SUM(total) AS revenue
    FROM walmart
    WHERE YEAR(STR_TO_DATE(date, '%d/%m/%y')) = 2022
    GROUP BY 1
),
revenue_2023 AS (
    SELECT branch, SUM(total) AS revenue
    FROM walmart
    WHERE YEAR(STR_TO_DATE(date, '%d/%m/%y')) = 2023
    GROUP BY 1
)
SELECT 
    ls.branch,
    ls.revenue AS last_year_revenue,
    cs.revenue AS cr_year_revenue,
    ROUND((ls.revenue - cs.revenue) / ls.revenue * 100, 2) AS rdr
FROM revenue_2022 AS ls
JOIN revenue_2023 AS cs ON ls.branch = cs.branch
WHERE ls.revenue > cs.revenue
ORDER BY 4 DESC
LIMIT 5;

Buckets each transaction into a shift using a CASE expression based on the hour of sale, then counts invoices per branch and shift — a quick read on when each location is busiest during the day.

  • A handful of branches saw double-digit revenue drops from 2022 to 2023. I caught that with a two-CTE year-over-year comparison — definitely something worth looking into more at the branch level.

Netflix Content Analysis

I was curious what Netflix's content catalog actually looks like under the hood, so I queried 8,807 titles in MySQL — looking at the movie vs. TV show split, ratings, which genres and countries dominate, and even flagging content by keywords in the description. One thing that kept coming up: fields like country, cast, and genre were stored as comma-separated values in a single column, so I had to split those out with string functions before I could count anything properly.

02
Top 5 countries by content volume

The country column often lists multiple co-production countries per title (e.g. "United States, Canada"). A numbers-CTE join splits each row into one entry per country before counting, so a title credited to three countries is counted once for each.

String Parsing
SELECT 
    TRIM(SUBSTRING_INDEX(SUBSTRING_INDEX(country, ',', n.n), ',', -1)) AS country,
    COUNT(*) AS total_content
FROM netflix
JOIN (SELECT 1 n UNION SELECT 2 UNION SELECT 3 UNION SELECT 4 UNION SELECT 5) n
    ON CHAR_LENGTH(country) - CHAR_LENGTH(REPLACE(country, ',', '')) >= n.n - 1
WHERE country IS NOT NULL AND country != ''
GROUP BY TRIM(SUBSTRING_INDEX(SUBSTRING_INDEX(country, ',', n.n), ',', -1))
HAVING country != ''
ORDER BY total_content DESC
LIMIT 5;

15 "Good" vs. "Bad" content by keyword CTE + CASE

A CTE tags every title as Bad if its description contains "kill" or "violence", otherwise Good, then counts titles in each bucket — a lightweight content-sensitivity flag based purely on description text. 

WITH new_table AS (
    SELECT *,
        CASE

            ELSE 'Good'
        END AS category
    FROM netflix
)
SELECT
    category,
    COUNT(*) AS total_content
FROM new_table
GROUP BY 1;
Key Takeaways
  • Movies outnumber TV shows by more than 2-to-1 (6,131 vs. 2,676), and TV-MA is the most common rating across both — which honestly makes sense given how much of Netflix's brand is built around adult-oriented content.
  • The US absolutely dominates by country of origin with ~3,683 titles — more than triple India's ~1,045, which came in second. And that's after splitting co-productions so every contributing country gets counted.
  • India's content on Netflix was heavily front-loaded — 2018 alone made up about 34% of all Indian titles ever added to the platform, and it's been declining since. That pattern made me curious about how Netflix's regional licensing deals have shifted over time.
  • The messiest part of this dataset was that country, cast, and genre were all stored as comma-separated strings in one column. Almost every "count by X" question required splitting those fields into individual rows first — good practice for dealing with non-normalized data.

Data Cleaning

A window function handled the problem: exact-duplicate admission records (same patient, same admission date, same discharge date — not legitimate readmissions, which differ by date) needed to be collapsed before any metric could be trusted.

—
Deduplicated view (vw_admissiondata)

ROW_NUMBER() partitioned on patient + admission + discharge date flags exact duplicates as anything after the first occurrence; every query below runs against this view rather than the raw table.

Window Fn
CREATE OR REPLACE VIEW vw_AdmissionData AS
WITH CleanData AS (
    SELECT *,
    ROW_NUMBER() OVER(PARTITION BY mrd_no, doa, dod ORDER BY mrd_no) AS Dup_NO
    FROM hdhi_admission_data
)
SELECT *
FROM CleanData
WHERE Dup_NO = 1 AND mrd_no IS NOT NULL;

Key Takeaways
  • About 7.7% of the records (1,217 out of 15,757) were exact duplicates — same patient, same admission and discharge dates. I caught them with a ROW_NUMBER() window function before running any calculations. Had I left them in, they would've quietly thrown off every count and average in the analysis.
  • The average length of stay came out to exactly 7 days, which makes sense for a cardiac specialty hospital — most patients aren't in and out same-day, they're there for monitoring or some kind of intervention.
  • Discharges took a noticeable dip on Sundays (1,486 total) — about 20% below the weekday average. That lines up with what you'd expect: hospitals tend to hold off on non-urgent discharges over the weekend when follow-up care and staffing are more limited.
