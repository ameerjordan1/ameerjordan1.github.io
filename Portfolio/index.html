Sentiment Analysis with NLTK & HuggingFace Transformers

This project applies Natural Language Processing techniques to classify sentiment in over 500,000 Amazon product reviews. Using both NLTK's VADER model and a HuggingFace RoBERTa transformer, review text is scored and compared across positive, neutral, and negative categories, with results visualized against star ratings.

Notebook
Results

Across 500 Amazon reviews, both models performed well but told different stories. VADER's compound scores trended in the expected direction — lower for 1-star reviews and higher for 5-star reviews — but showed notable overlap between mid-range ratings (2–4 stars), where sarcasm and nuanced language challenged its rule-based approach. The RoBERTa transformer consistently produced sharper, more confident sentiment separations, particularly on reviews where the tone contradicted the literal word choice — something VADER's bag-of-words method struggled to detect.

Fake News Detection System

This project builds a machine learning pipeline that classifies news articles as Fake or Real based on their text content. The workflow covers loading and merging labeled fake/real news datasets, cleaning and preprocessing article text, vectorizing it with TF-IDF, and training multiple classification models to compare performance.

Notebook
Results

All four models performed exceptionally well on the held-out test set of over 11,000 articles. Logistic Regression and Random Forest both achieved roughly 98.8% accuracy, while the Decision Tree and Gradient Boosting classifiers edged slightly higher at around 99.5%, with near-perfect precision, recall, and F1-scores across both classes. On manual testing with unseen news articles, all four models agreed consistently — correctly flagging fabricated stories as Fake News and legitimate reporting as Real News.

Walmart Sales Analysis

An exploratory analysis of multi-branch retail transaction data using MySQL — covering payment behavior, category performance, peak shopping windows, and year-over-year revenue trends across branches. SQL was used for data extraction and aggregation; the Jupyter notebook below covers further cleaning, visualization, and summary statistics.

Business Questions & Queries

Four queries chosen to show a range of techniques — from simple aggregation to window functions and multi-CTE year-over-year comparisons.

01
Payment method breakdown

How many transactions and units sold per payment method — a quick read on which channels customers actually use.

Aggregation
SELECT 
    payment_method,
    COUNT(*) AS no_payments,
    SUM(quantity) AS no_qty_sold
FROM walmart
GROUP BY payment_method;

02
Top-rated category per branch

Ranks categories by average rating within each branch, then keeps only the #1 — surfacing each location's standout department.

Window Fn
SELECT *
FROM (
    SELECT
        branch,
        category,
        AVG(rating) AS avg_rating,
        RANK() OVER (PARTITION BY branch ORDER BY AVG(rating) DESC) AS `rank`
    FROM walmart
    GROUP BY 1, 2
) AS ranked
WHERE `rank` = 1;

03
Busiest day per branch

Converts the stored date string to a weekday name, counts transactions per day, and ranks to find each branch's peak day.

Window Fn
SELECT *
FROM (
    SELECT
        branch,
        DAYNAME(STR_TO_DATE(date, '%d/%m/%y')) AS day_name,
        COUNT(*) AS no_transactions,
        RANK() OVER (PARTITION BY branch ORDER BY COUNT(*) DESC) AS `rank`
    FROM walmart
    GROUP BY 1, 2
) AS ranked
WHERE `rank` = 1;

04
Year-over-year revenue decline

Two CTEs total 2022 and 2023 revenue per branch, then join them to compute a revenue decrease ratio and surface the 5 branches that dropped the most.

Multi-CTE
WITH revenue_2022 AS (
    SELECT branch, SUM(total) AS revenue
    FROM walmart
    WHERE YEAR(STR_TO_DATE(date, '%d/%m/%y')) = 2022
    GROUP BY 1
),
revenue_2023 AS (
    SELECT branch, SUM(total) AS revenue
    FROM walmart
    WHERE YEAR(STR_TO_DATE(date, '%d/%m/%y')) = 2023
    GROUP BY 1
)
SELECT 
    ls.branch,
    ls.revenue AS last_year_revenue,
    cs.revenue AS cr_year_revenue,
    ROUND((ls.revenue - cs.revenue) / ls.revenue * 100, 2) AS rdr
FROM revenue_2022 AS ls
JOIN revenue_2023 AS cs ON ls.branch = cs.branch
WHERE ls.revenue > cs.revenue
ORDER BY 4 DESC
LIMIT 5;

06 Rating spread by city and category Aggregation

Minimum, maximum, and average rating for every city/category combination — useful for spotting categories with inconsistent customer experience. 

SELECT 
    city,
    category,
    MIN(rating) AS min_rating,
    MAX(rating) AS max_rating,
    AVG(rating) AS avg_rating
FROM walmart
GROUP BY 1, 2;
07 Profit by category Derived Metric

Calculates total profit per category as revenue × profit margin, ranked from most to least profitable — a quick view of which departments actually drive margin, not just revenue. 

SELECT 
    category, 
    SUM(total) AS total_revenue,
    SUM(total * profit_margin) AS profit
FROM walmart
GROUP BY 1
ORDER BY profit DESC;
08 Preferred payment method per branch CTE + Window Fn

A variant of the top-category query — ranks payment methods by transaction count within each branch and keeps the most-used one. 

WITH cte AS (
    SELECT 
        branch,
        payment_method,
        COUNT(*) AS total_trans,
        RANK() OVER (PARTITION BY branch ORDER BY COUNT(*) DESC) AS `rank`
    FROM walmart
    GROUP BY 1, 2
)
SELECT * 
FROM cte
WHERE `rank` = 1;
09 Invoices by shift (morning/afternoon/evening) Conditional Logic

Buckets each transaction into a shift using a CASE expression based on the hour of sale, then counts invoices per branch and shift — a quick read on when each location is busiest during the day. 

SELECT
    branch,
    CASE
        WHEN HOUR(time) < 12 THEN 'MORNING'
        WHEN HOUR(time) BETWEEN 12 AND 17 THEN 'AFTERNOON'
        ELSE 'EVENING'
    END AS shift,
    COUNT(*) AS num_invoices
FROM walmart
GROUP BY branch, shift
ORDER BY branch, num_invoices DESC;
Key Takeaways
  • Credit card and e-wallet payments together account for the large majority of transactions and units sold — cash is a minority channel and could shape staffing or terminal allocation decisions.
  • Top-rated categories vary meaningfully by branch, suggesting location-specific merchandising or local demand differences rather than a single chain-wide "best" category.
  • Several branches showed double-digit revenue declines from 2022 to 2023, flagged via a simple two-CTE join — a pattern worth investigating further at the branch-operations level.
